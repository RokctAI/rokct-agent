# Platform Comparison: GoClaw vs. Hermes Agent

This directory contains a detailed technical comparison between the current **GoClaw fork** and **Hermes Agent**. Each file focuses on a specific comparison point to facilitate focused discussion.

## Comparison Topics

1.  [**Core Architecture & Language**](01_core_architecture.md) — Binary vs. Python, Efficiency vs. Ecosystem.
2.  [**Multi-Tenancy & Data Isolation**](02_multi_tenancy.md) — PostgreSQL vs. SQLite, Scaling vs. Personal use.
3.  [**Learning Loop & Memory**](03_learning_loop.md) — Passive storage vs. Active self-improvement.
4.  [**Skills & Extensibility**](04_skills_extensibility.md) — Progressive disclosure and tool creation.
5.  [**Messaging Channels & WhatsApp**](05_messaging_whatsapp.md) — Voice support and general channels.
6.  [**Coding Engine Capabilities**](06_coding_engine.md) — Intent routing and architectural mapping.
7.  [**Multi-Agent Orchestration**](07_multi_agent_orchestration.md) — Teams vs. Ephemeral Subagents.
8.  [**WhatsApp Bridge Deep Dive**](08_whatsapp_bridge_deep_dive.md) — Technical comparison of bridge implementations.

## Summary Recommendation

For the **Life/Career/Legacy Manager** vision, **Hermes Agent** is the recommended foundation due to its active learning loop and Python-based extensibility. However, GoClaw remains the stronger choice for large-scale **Multi-Tenant SaaS** infrastructure.
